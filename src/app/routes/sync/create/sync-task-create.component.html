同步路径

源表 字段映射 目标表

策略要点

  • 全量适合初始化，增量适合持续同步。
  • 增量任务建议设置稳定递增的游标字段。
  • upsert 需要冲突字段与目标表索引匹配。

映射示例

{"source":"created_at","target":"ts"}
{"target":"source_type","default":"mysql"}

字段映射写法

  • source 表示源字段，target 表示写入目标字段。
  • default 可写入固定值，适合补充来源、类型或租户标识。
  • transform 可用于时间、数字等格式转换，保存前先校验配置。

模式选择

全量同步 适合首次导入、结构调整后重建目标表，执行时会按过滤条件重新读取源表。
增量同步 适合定时任务，需要稳定递增的游标字段，例如自增 id 或写入时间。
批次大小 默认 1000 更稳妥；大表可适当调大，但要观察数据库连接和写入压力。

提交前检查

  • 先读取源表和目标表字段，确认字段名和类型匹配。
  • 使用过滤条件时，建议先预估源数据量，避免误同步过多数据。
  • 启用定时前先手动执行一次，确认游标推进和写入结果正常。
