备份包内容

每次备份会生成一个压缩包，包含清单、表结构和按行导出的数据文件。

适用场景

  • 上线、迁移或批量修改数据前留存快照。
  • 为问题排查、数据回滚或审计保留可追溯记录。

执行说明

  • 备份为异步任务，提交后可离开当前页面。
  • 成功后生成 zip 文件，可在备份历史中下载。
  • 连接失败或权限不足时，系统会记录失败原因。

成功率建议

  • 确认账号有读取表结构和数据的权限。
  • TDengine 大表优先设置 timeout，例如 5m。
  • MySQL 慢查询建议设置 readTimeout 和 writeTimeout。
  • 遇到偶发 EOF 或连接重置，可保留 2-3 次批次重试。
